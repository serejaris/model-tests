# examples/ — правила каталога

Сюда кладутся **артефакты от прогонов этого теста на разных моделях** —
по одному под-каталогу на каждую модель (`examples/<model-slug>/`).

## Чего здесь НЕ должно быть

- **Готового решения от какой-либо одной модели, выложенного как
  «образец»**. Если модель уже проходила этот тест, и артефакт живёт
  в её основном треке (`../<model>/<test>/…`) — **сюда его дублировать
  не нужно**. Здесь лежат только результаты прогонов, а не
  подсказки для будущих моделей.
- Снапшотов, tutorials, «как это делать по шагам». Это загрязняет
  priors следующего прогона.

## Как добавлять

1. Прогнать тест на новой модели в отдельной сессии.
2. Положить артефакт в `examples/<model-slug>/`.
3. Если это фронтенд — снять скриншоты в
   `examples/<model-slug>/screenshots/`.
4. Написать `examples/<model-slug>/README.md` с моделью, оценкой по
   criteria, smoke-evidence.
5. Создать child issue от трека модели по правилам `manager`.

## Сейчас в каталоге

рамках чистого прогона (без подсказок).
