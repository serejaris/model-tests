# Критерии оценки — 3D shooter game

Эти критерии используются, чтобы сравнить, как разные модели
выполняют один и тот же тест. Чем больше критериев закрыто — тем
лучше артефакт.

## Структурные (must-have)

- [ ] **Запускаемый Vite-проект** (или эквивалент) с `package.json`,
      `index.html`, исходниками в `src/`.
- [ ] **`npm run build` без ошибок** и **без предупреждений уровня
      error**.
- [ ] **`npm run preview` (или `dev`) поднимает сервер** и страница
      открывается в браузере без `localhost:0`-ошибок.
- [ ] **Headless smoke** (`smoke.mjs` или эквивалент) проходит
      проверки и завершается с exit 0.
- [ ] **Самодостаточный** — никаких внешних ассетов (никаких
      `.glb`, `.png`, `.jpg`; всё процедурно через `BoxGeometry`,
      `CylinderGeometry`, `PlaneGeometry`, и т.п.).
- [ ] **Без `.env`-файлов и приватных ключей** в репо.

## Содержательные (must-have)

- [ ] **3D-сцена** с полом, стенами, какими-то объектами (ящики,
      столбы, бочки, лестницы — что угодно, что делает пространство
      не плоским).
- [ ] **FPS-управление игрока**: WASD + мышь (pointer-lock), прыжок
      опционально, спринт опционально.
- [ ] **Стрельба** с raycast-логикой попадания (не magic-пиксель-хантинг
      в центре экрана без геометрии).
- [ ] **Хотя бы один противник** с ИИ, который реагирует на игрока
      (патруль, погоня, атака — что угодно, но не «статуя»).
- [ ] **HUD**: минимум HP игрока + счётчик чего-то (патроны или
      enemies left).
- [ ] **Win/lose-условие** и overlay, который показывает исход.
- [ ] **Играбельно** в реальном браузере: не вылетает, не зависает,
      курсор-лок работает, мышь влияет на камеру.

## Антипаттерны (минусы, если найдены)

- [ ] **Нет заглушек TODO** в ключевых местах (`fireWeapon()` пустой,
      ИИ — пустая функция). Если TODO, он должен быть в плане, а
      не в коде.
- [ ] **Нет магических чисел** без комментариев (например, скорость
      5.5 — ок, 0.0837 — лучше с пояснением).
- [ ] **Нет «как в Counter-Strike»** (или любой другой игре) формулировок
      в README без конкретики. «Как в CS» — это маркетинг, не ТЗ.
- [ ] **Нет загрузки внешних .glb/текстур** (мы не должны видеть
      `loadAsync('models/something.glb')` в коде).
- [ ] **Нет `console.log`-спама** в коде продакшна.

## Визуальные / качественные (бонус)

- [ ] **Low-poly эстетика** (BoxGeometry + `MeshLambertMaterial` +
      `flatShading`) — в духе CS 1.6 / Quake.
- [ ] **Импакт-эффекты**: muzzle flash, tracer, декали или искры при
      попадании.
- [ ] **Full-auto очередь** (удержание LMB даёт серию выстрелов, не
      одиночный).
- [ ] **Анимация смерти** противника (плавный наклон, не мгновенный
      «despawn»).
- [ ] **Hit reaction** (бот визуально реагирует на попадание —
      наклон / эмиссия / тряска).
- [ ] **Разнообразные состояния** (старт / gameplay / победа / поражение)
      с визуально различимыми экранами.
- [ ] **Звуковая обратная связь** (хотя бы WebAudio-синтез, без
      аудио-файлов).

## Скриншоты (обязательны)

- Стартовый экран.
- Игровой кадр в середине матча.
- Момент стрельбы.
- Финальный экран (victory или defeat).

## Как оценивать

- 5/5 — все структурные + все содержательные + 0 антипаттернов +
  ≥3 бонусных.
- 4/5 — все структурные + ≥6 содержательных + 0 антипаттернов + ≥2
  бонусных.
- 3/5 — все структурные + ≥5 содержательных.
- ≤2 — тест не пройден, нужен ретрай или другой промпт.

## Smoke-проверка (минимум)

`smoke.mjs` в `examples/<model>/` должен проверить через headless
Puppeteer:

1. Страница рендерится без `pageerror`.
2. WebGL-контекст создан.
3. `ammo` уменьшается при имитации выстрела.
4. `hp` игрока уменьшается, когда бот видит его и стреляет.
5. После `takeDamage` на всех ботах достижимо victory-условие
   (overlay `VICTORY`).
6. 0 `console.error` в headless.

## Чего этот тест НЕ измеряет

- Качество графики / текстур (low-poly по дизайну, PBR не
  предполагается).
- Скорость работы. Может лагать на 10 FPS — это не минус для
  теста.
- Реалистичность стрельбы, баллистику, отдачу как физику.
- Мультиплеер.
- Адаптивный ИИ (боты могут быть тупыми — достаточно, чтобы
  реагировали).
