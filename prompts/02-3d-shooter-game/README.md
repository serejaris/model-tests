# Тест 02 — 3D shooter game

Минимально приемлемый 3D-шутер от первого лица в браузере. На входе
— дословный user-промпт из трёх слов, на выходе — запускаемый
Vite-проект с процедурной графикой, FPS-управлением, raycast-
стрельбой и хотя бы одним противником с ИИ.

## Вход

Дословный промпт в [`prompt.md`](./prompt.md). Контекст — в том же
файле.

## Ожидаемый артефакт

Vite (или эквивалент) проект с:
- `package.json`, `index.html`, `src/…`
- без внешних ассетов
- headless smoke, проходящий критерии из `criteria.md`

## Критерии оценки

В [`criteria.md`](./criteria.md). Единый чеклист для всех моделей.

## Примеры

- [`examples/minimax-m3/`](./examples/minimax-m3/) — как этот тест
  прошла модель `opencode-go/minimax-m3`. Self-contained копия
  Vite-проекта.

## Как запустить тест на новой модели

1. Открой новую сессию с целевой моделью.
2. Скопируй дословный текст из `prompt.md` и отправь модели.
3. Модель, скорее всего, начнёт спрашивать детали. **Не отвечай** —
   дай ей самой принять разумные дефолты. Если она зависает без
   ответа, только тогда задай уточняющий вопрос.
4. Получи артефакт (запускаемый Vite-проект).
5. Прогони `criteria.md` по результату. Зафиксируй оценку.
6. Положи артефакт в `examples/<model-slug>/`.
7. Сделай скриншоты через `smoke.mjs`, положи в
   `examples/<model-slug>/screenshots/`.
8. Напиши `examples/<model-slug>/README.md`: модель, оценка,
   smoke-результат, ссылка на issue.
9. Создай child issue от трека модели по правилам `manager`.

## Чего этот тест НЕ измеряет

- Качество графики. Low-poly — это стиль, не ограничение.
- Скорость работы.
- Реалистичность стрельбы.
- Мультиплеер.
