# prompts — общие тестовые промпты

Эта папка — каталог **переиспользуемых тестов**, не привязанных
к конкретной модели. Каждый промпт — это самостоятельный мини-тест,
который можно гонять на разных моделях и сравнивать результаты.

## Зачем отделять от `<модель>/prompts/`

В каждой модели (`minimax-m3/`, `claude-opus-4/` и т.п.) лежит своя
`prompts/` — там **реальные user-промпты, на которые модель
отвечала в ходе трека**. Это исторический журнал.

А здесь — **шаблоны тестов**, которые можно применить к любой
модели, чтобы собрать калиброванный результат. Это не «промпт,
который юзер написал», а «промпт, который мы как тестеры написали,
чтобы модель сделала конкретную вещь».

## Структура

```
prompts/
├── README.md
└── <NN-slug>/
    ├── prompt.md          # дословный текст промпта + контекст
    ├── criteria.md        # как оценивать артефакт
    ├── README.md          # что за тест, как переиспользовать
    └── examples/
        └── <model-slug>/
            ├── <артефакт>     # то, что выдала модель
            ├── screenshots/    # если фронтенд
            └── README.md       # от какой модели, как открыть, smoke
```

## Как запустить тест на новой модели

1. Прочитай `prompt.md` — там дословный текст, который надо
   отправить модели **без переписывания**.
2. Прочитай `criteria.md` — там критерии, по которым будешь
   оценивать результат.
3. Положи артефакт в `examples/<model-slug>/`.
4. Если это фронтенд — сними скриншоты, в README теста должна
   появиться секция «Скриншоты» со встроенными PNG.
5. Сделай smoke-проверку, задокументируй в `examples/<model-slug>/README.md`.
6. Создай issue по правилам `manager` (эпик на трек модели + child
   issue на тест) — и обнови body ссылкой на артефакт и smoke.

## Чего этот каталог НЕ делает

- Не заменяет PRD конкретного теста. PRD — это «что мы строим
  внутри одного проекта». Здесь — наоборот: «вот идентичная
  задача, дай мне сравнить, как разные модели её решат».
- Не накапливает приватные данные, токены, реальные API-ключи.
  Примеры — это публичные артефакты.
- Не подменяет `<модель>/prompts/` — там живые сессии, здесь
  шаблоны для воспроизводимого сравнения.
