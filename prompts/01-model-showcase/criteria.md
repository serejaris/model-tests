# Критерии оценки — model-showcase

Эти критерии используются, чтобы сравнить, как разные модели
выполняют один и тот же тест. Чем больше критериев закрыто —
тем лучше артефакт.

## Структурные (must-have)

- [ ] **Один HTML-файл**, без внешних зависимостей (CSS, JS, SVG
  инлайнятся, шрифты — только system).
- [ ] **Размер файла < 200 КБ** без base64-картинок. Один HTML
  значит один файл, никаких `assets/img/...png`.
- [ ] **Самодостаточный** — открывается в браузере с `file://`,
  работает без сети.
- [ ] **Mobile-friendly** — на 360×640 ширины страница не
  разваливается (overflow не появляется).

## Содержательные (must-have)

- [ ] **Рассказывает про саму модель** — что это за модель, от
  какой компании/команды, какая версия/семейство.
- [ ] **Конкретные технические особенности** — контекст, тип
  задач, сильные/слабые стороны. Не «AI-революция», а числа и
  факты.
- [ ] **Есть диаграмма или схема** — inline SVG, минимум одна.
- [ ] **Есть интерактивный виджет** — tabs, accordion, comparison
  slider, playground, что-то, что реагирует на клик/наведение.
- [ ] **Есть примеры кода** — реальные сниппеты на JS/Python/etc,
  подсвеченные, с копированием или хотя бы с читаемой типографикой.

## Антипаттерны (минусы, если найдены)

- [ ] **Нет ИИ-слоп-фраз** в основном тексте. Список «стоп-слов»
  ниже.
- [ ] **Нет «Lorem ipsum»**, нет заглушек «Coming soon», «TBD».
- [ ] **Нет «AI is changing the world»** общего вида — никаких
  риторических вопросов без ответа.
- [ ] **Нет кнопок-пустышек** — каждая CTA либо реально ведёт
  куда-то, либо явно помечена как «демо».

## Стоп-слова ИИ-слопа (если в тексте ≥3 — минус к оценке)

- «революционн» / «прорыв» / «cutting-edge» / «state-of-the-art»
- «раскройте потенциал» / «unlock the power» / «transform your»
- «в современном мире» / «в эпоху AI» / «в эпоху перемен»
- «уникальная возможность» / «узнайте больше» / «discover the»
- «безграничные возможности» / «unleash» / «empower»
- «by leveraging» / «in today's fast-paced world» / «in the era
  of»
- Заголовки, состоящие из одного прилагательного без существительного
  («Быстрый. Умный. Ваш.», «Smart. Fast. Yours.»)
- «Революция в …», «Будущее уже здесь», «Future is now»

## Визуальные (бонус)

- [ ] SVG-диаграмма **архитектуры** пайплайна (input → model →
  output), а не просто декоративная иконка.
- [ ] **Comparison table** с другими моделями (честная, не
  самовосхваляющая).
- [ ] **Playground** с кодом и кнопкой «Run», эмулирующей ответ
  модели.
- [ ] **Timeline** или **roadmap** с реальными вехами.
- [ ] **Тёмная тема** или **две темы с переключателем**.

## Как оценивать

- 5/5 — все структурные + все содержательные + 0 стоп-слов + ≥3
  бонусных.
- 4/5 — все структурные + ≥4 содержательных + ≤1 стоп-слово + ≥1
  бонус.
- 3/5 — все структурные + ≥3 содержательных.
- ≤2 — тест не пройден, нужен ретрай или другой промпт.

## Smoke-проверка

`smoke.mjs` (в каждом `examples/<model>/`) должен:

1. Запустить локальный HTTP-сервер на порту 4100+ для папки с
   `index.html`.
2. Открыть страницу headless Chromium.
3. Сделать скриншоты: hero, scroll-down, взаимодействие с виджетом
   (например клик на tab).
4. Проверить, что в DOM есть хотя бы один `<svg>` и один
   интерактивный элемент (`button`, `details`, `tab`).
5. Проверить, что страница не падает в консоли.
