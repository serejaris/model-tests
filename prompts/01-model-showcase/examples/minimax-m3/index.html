Возможности

M3 проектировался не как максимально большая «говорящая голова», а как рабочая лошадка для пайплайнов. Четыре области, где это проявляется ярче всего.

Код, который потом читают и поддерживают

Модель уверенно держит контекст проекта, читает несколько файлов подряд, делает рефакторинги, не теряя согласованности с ранее написанным кодом. Хорошо работает с типизированными языками (TypeScript, Python c type hints, Rust).

  • Пишет код, который предсказуемо ведёт себя в edge-кейсах
  • Предпочитает явные имена и понятные структуры данных
  • В long-running сессиях помнит, какие конвенции выбраны раньше
  • Возвращается к точным именам функций и API, упомянутых выше по диалогу

Нативный tool use

M3 обучен вызывать инструменты структурированно: модель возвращает JSON-описание вызова, рантайм выполняет, результат возвращается в контекст. Поддерживаются параллельные вызовы, условные вызовы и многошаговые цепочки.

  • Чистый формат tool calls, без обёрток из "let me..." мусора
  • Умеет отказаться от вызова, если не уверен
  • Работает со схемами JSON Schema произвольной сложности
  • Не пытается парсить результат инструмента как «инструкцию»

Длинный контекст без «забывания»

Документ, репозиторий, длинная транскрипция — модель способна удерживать связность по всему контексту. Но это не бесплатно: качество постепенно деградирует к хвосту, и это стоит учитывать в дизайне промптов.

  • Работает с system-промптами на 5–10 КБ без потери инструкций
  • Может вернуться к деталям, упомянутым в начале длинного документа
  • Чем ближе к началу/концу контекста — тем выше точность; середина слабее (как у всех трансформеров)

Многоходовые диалоги

Хорошо работает в диалоге с пользователем и в длинных цепочках рассуждений. Может сам себя поправлять, отслеживать противоречия в собственных утверждениях и возвращаться к более ранним решениям, когда получает обратную связь.

  • Спокойно относится к «нет, переделай» — не пытается защищать старое
  • Не теряет thread на 20+ ходов
  • Помнит, какие инструменты уже вызывал, не повторяет одинаковые вызовы

Архитектура пайплайна инференса

M3 — декодер-онли трансформер. Диаграмма ниже показывает, что происходит с одним user-turn'ом от получения токенов до выдачи ответа. Стрелки — потоки данных, цвета — разные стадии.

Tool-use loop

    

      Минимальный цикл взаимодействия модели с инструментами. M3
      возвращает структурированный вызов, рантайм исполняет, результат
      возвращается в контекст, и так до тех пор, пока модель не решит,
      что ответ готов.
    


    
  






  

    
С инструментами vs без

    

      Один и тот же запрос, два режима. Двигайте разделитель, чтобы
      увидеть разницу.
    


    

      

        

          
Без tool use

          

            Модель отвечает «из того, что знает». Если в её обучении
            не было точной цифры или свежего факта — она либо
            откажется отвечать, либо выдаст правдоподобную отсебятину.
            На вопрос «какой курс доллара сегодня» без инструмента
            у M3 нет шансов дать правильный ответ.
          

        

        

          
С tool use

          

            M3 возвращает структурированный вызов get_exchange_rate("USD"),
            рантайм исполняет, кладёт результат в контекст, и модель
            продолжает уже с актуальной цифрой. Источник указан, факт
            проверяемый, ответ привязан ко времени.

Честно об ограничениях

    

      M3 — рабочая лошадка, а не волшебная палочка. Что нужно
      учитывать при дизайне пайплайнов.
    


    

      

        
Галлюцинации на длинных фактах

        
На неочевидных числовых фактах (даты, номера версий, цены, ID) ошибается. Сверяйте критичные данные с источником или заставляйте модель вызывать инструмент для их получения.

      

      

        
Latency на длинном контексте

        
Первый токен ответа приходит с задержкой, пропорциональной размеру контекста. Для real-time диалогов держите контекст компактным, выносите старые сообщения в RAG.

      

      

        
Стоимость

        
Чем больше контекст — тем дороже запрос. На длинных диалогах используйте summary старых ходов или скользящее окно.

      

      

        
Cutoff знаний

        
Базы знаний M3 заканчиваются на январе 2026. События после — требуют либо RAG, либо tool calls, либо честного «не знаю».
