# model-tests

Публичный каталог **тестовых треков по моделям**. Одна папка — одна
AI-модель под тестом. Внутри папки модели сырые промпты и
получившиеся артефакты тестов лежат рядом, чтобы любой мог
воспроизвести тест ровно в том виде, в каком он был прогнан.

## Зачем это нужно

Когда я оцениваю модели, хочу, чтобы **промпт**, **код**, который
модель выдала, и **доказательства smoke-проверки** жили в одном
месте — публично, доступные для поиска и воспроизводимые. Этот
репозиторий — именно такое место.

## Структура

```
<имя-модели>/
├── README.md             # что за модель, что тестируем
├── prompts/<NN-slug>/    # сырые user-промпты, дословно
└── <имя-теста>/          # самодостаточный артефакт теста
    ├── README.md         # что это, как запустить, что измерили
    └── …
```

Полные правила — в [AGENTS.md](./AGENTS.md).
