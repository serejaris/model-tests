# model-tests

Публичный каталог **тестовых треков по моделям**. Одна папка — одна
AI-модель под тестом. Внутри папки модели сырые промпты,
получившиеся артефакты тестов, **PRD** и **скриншоты** живут
рядом, чтобы любой мог воспроизвести тест ровно в том виде, в
каком он был прогнан.

## Зачем это нужно

Когда я оцениваю модели, хочу, чтобы **промпт**, **код**, который
модель выдала, **PRD**, по которому модель работала, и
**доказательства smoke-проверки** жили в одном месте — публично,
доступные для поиска и воспроизводимые. Этот репозиторий —
именно такое место.

## Что лежит в каждом тесте

| Артефакт | Зачем |
|---|---|
| `prompts/<NN-slug>/prompt.md` | дословный user-промпт, чтобы тест можно было повторить |
| `PRD.md` | цели, scope, метрики успеха, ограничения, non-goals |
| `src/…` | исходники, которые модель выдала по промпту |
| `screenshots/…` | визуальные кадры UI / игры (только для фронтенда) |
| `README.md` | как запустить, что измерили, известные проблемы |
| `smoke.mjs` или эквивалент | автоматизированная проверка теста |

## Структура

```
<имя-модели>/
├── README.md             # что за модель, что тестируем
├── prompts/<NN-slug>/    # сырые user-промпты, дословно
└── <имя-теста>/          # самодостаточный артефакт теста
    ├── README.md
    ├── PRD.md            # обязателен для нетривиальных тестов
    ├── screenshots/      # обязателен для фронтенда
    └── …
```

Полные правила — в [AGENTS.md](./AGENTS.md).

## Трекинг

Каждый тест также задокументирован как GitHub issue в
[`serejaris/model-tests`](https://github.com/serejaris/model-tests/issues)
по правилам skill `manager` — один epic на модель, один child-issue
на тест, всё прокинуто в `ris © corp` Project #4. До начала работы
по тесту в issue лежит PRD-блок; в конце — скрины и smoke-evidence.
