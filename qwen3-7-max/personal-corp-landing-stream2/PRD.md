# PRD — Personal Corp Stream 2 Landing

## 1. Контекст и цель

Sales-лендинг для второго потока программы **Personal Corp** (Личная Корпорация).

Программа — 8-недельная мастерская, где человек превращает хаотичное использование AI в личную операционную систему: штаб, правила, отделы, повторяемые процессы, видимые артефакты и weekly operating cycle.

Первый поток (10 человек, 39 000 ₽, старт 20.04.2026) завершён 15.06.2026. Есть реальные результаты, отзывы и ретро на 244 вопроса студентов.

**Цель лендинга:** перевести прогретый трафик (после вебинара 17.06, из TG-канала, комьюнити) в переход в бот `@hashslash_bot` для оплаты и старта потока 2.

**Параметры потока 2:**
- Старт: 23 июня 2026 (вторник)
- Занятия: по вторникам 19:00 МСК
- 8 встреч: 23.06 → 11.08.2026
- Формат: живые встречи в Zoom, 1.5 часа

## 2. Scope (in / out)

### In
- Одностраничный sales-лендинг (single HTML file with inline CSS/JS)
- Секции: Hero, Problem, Transformation, Who, Self-check, Not fit, Graduation, Program (8 недель), Format, Pricing, Author + proof, FAQ, Final CTA
- Данные программы из `weeks.js` (уже подготовлены для потока 2)
- Proof layer из ретро потока 1 (реальные отзывы, результаты)
- Mobile-first responsive
- CTA → deeplink в бот

### Out
- React/Vite/сборщик — нужен single-file HTML без build step
- Оплата на странице (только перевод в бот)
- Email-сбор, формы, аналитика на странице
- LMS, knowledge base, program source of truth
- Team-format redirect (для команд — отдельный формат)

## 3. Пользовательские сценарии

1. **После вебинара:** человек кликает ссылку из follow-up рассылки → видит Hero с обещанием → скроллит к программе → читает 8 недель → кликает CTA → попадает в бот для оплаты.
2. **Из TG-канала:** читает пост Риса → переходит на лендинг → быстро сканирует "для кого" → находит себя → читает proof → кликает CTA.
3. **Repeat buyer (из Кружка):** видит знакомое имя → проверяет отличие от Кружка → читает self-check → понимает что пора → CTA.
4. **Скептик:** читает "кому не подойдёт" → уважает честность → либо уходит, либо остаётся с большим доверием.

## 4. Метрики успеха

- Лендинг рендерится без ошибок в Chrome, Safari, Firefox (desktop + mobile)
- Single-file HTML < 200 KB (без внешних зависимостей кроме шрифтов)
- Все 8 недель отображаются с корректными датами потока 2
- Все CTA ведут на корректный deeplink
- Lighthouse Performance ≥ 90
- Визуально: professional, clean, не AI-slop

## 5. Технические ограничения

- **Single HTML file** — без React, без Vite, без npm
- Tailwind CSS через CDN (как existing `index.html` в teach-vibecoding)
- Google Fonts (Inter или аналогичный sans-serif)
- Inline JS для минимальной интерактивности (accordion, countdown)
- Палитра: zinc/neutral base + один accent color
- Язык: русский

## 6. Non-goals

- Не клон существующего React-лендинга — это отдельный артефакт для model-tests
- Не продаёт другие продукты Риса
- Не является source of truth для программы
- Не заменяет workshop deck или бот
- Не раскрывает методологию полностью

## 7. План работы

### MVP (что делаем)
1. Каркас HTML с Tailwind CDN + Google Fonts
2. Hero: заголовок, обещание, дата старта, CTA
3. Problem: 4 боли из course-design.md
4. Transformation: дуга "вайбкодер → оператор → CEO"
5. Who: 4 сегмента аудитории
6. Self-check / Not fit: два блока самоотбора
7. Graduation: 3 ключевых outcomes
8. Program: 8 карточек недель из weeks.js данных
9. Format: формат участия
10. Pricing: цена и старт
11. Author + proof: авторский блок с proof points из потока 1
12. FAQ: 5-7 ключевых возражений
13. Final CTA: финальный перевод в бот

### Готово когда
- HTML валидный, рендерится без ошибок
- Все секции на месте, данные корректные
- Mobile responsive
- Скриншоты сняты

### Риски и план Б
- Объём контента большой → приоритизировать Program и CTA, остальные секции упростить
- Tailwind CDN медленный → fallback на inline styles
