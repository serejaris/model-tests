# PRD — model-showcase (minimax-m3)

## Контекст и цель

Общий тест `prompts/01-model-showcase/` — одностраничный сайт-визитка
про саму модель, на которой прогоняется. Этот прогон — первый
чистый заход `minimax-m3` на этом тесте: `examples/` пуст, чужих
решений не подсматриваю.

Цель артефакта — **честная самопрезентация**:

- что за модель (кто, какая версия, какая компания);
- что умеет делать хорошо (наблюдаемое поведение, а не маркетинг);
- где границы (что не умеет, чего не знает);
- как устроена под капотом в общих чертах (диаграмма pipeline).

Аудитория — люди, которые **сравнивают модели** (см. README
репозитория). Поэтому артефакт публичный, самодостаточный, один
HTML, открывается с `file://` без сети.

Почему сейчас: тест уже описан в `prompts/01-model-showcase/`,
`criteria.md` готов, шкала оценки 0–5 ясна. Брать в работу.

## Scope (in / out)

**In scope (MVP):**

1. Один `index.html` ≤ 200 КБ, без внешних ассетов, без base64-картинок.
2. Inline `<svg>` архитектурная диаграмма (input → processing → output).
3. Минимум 1 интерактивный виджет с реакцией на клик/ховер
   (план: tabs с примерами задач).
4. Минимум 1 playground со сниппетом кода и кнопкой «Run»,
   эмулирующей типичный ответ модели.
5. Блок «capabilities vs limitations» — честная таблица, без
   самовосхваления, с конкретными оговорками.
6. Минимум 1 timeline / roadmap (M1 → M2 → M3) с реальными
   вехами компании и модели.
7. Mobile-friendly (360×640 без горизонтального скролла).
8. Светлая и тёмная тема, переключатель.
9. Smoke-проверка Playwright/Puppeteer в `smoke.mjs`:
   - поднять локальный http-сервер;
   - открыть страницу;
   - проверить наличие `<svg>`, кнопок, отсутствие ошибок в консоли;
   - сделать скриншоты (hero, scroll, активный tab).

**Out of scope:**

- Реальные API-вызовы к модели (артефакт статический, без сети).
- Бенчмарки с цифрами, которые я не могу верифицировать. Если нужны
  числа — беру **наблюдаемые** характеристики (например, наличие
  thinking-режима), а не синтетические «MMLU 87.4%».
- Внешние шрифты, CDN, иконки-паки. Всё inline.
- Аналитика, cookie, трекеры.
- Серверный рендеринг, build-step. Один HTML открывается двойным
  кликом.

## Пользовательские сценарии

1. **«Кто ты?»** — посетитель скроллит от hero, читает секцию
   «О модели», получает название, компанию, cutoff, целевое
   назначение. За 30 секунд понятно, что это за сущность.
2. **«Что ты умеешь?»** — посетитель открывает вкладку в tabs
   («Код», «Структура», «Длинный контекст»), смотрит живой пример
   с подсветкой и кнопкой «Запустить». Сам нажимает — видит
   эмулированный ответ.
3. **«А в чём ты слаб?»** — посетитель листает к честной таблице
   capabilities vs limitations. Видит «нет доступа в сеть»,
   «знания до Jan 2026», «может галлюцинировать численные
   значения». Понимает границы.
4. **«Как ты устроен?»** — посетитель смотрит на inline-SVG
   диаграмму pipeline: prompt → tokenize → transformer blocks →
   sampling → ответ. С подписями, без магии.

## Метрики успеха

Из `criteria.md` теста:

- 5/5 — все 4 структурных + все 5 содержательных + 0 стоп-слов
  + ≥3 бонусных.
- 4/5 — все 4 структурных + ≥4 содержательных + ≤1 стоп-слово
  + ≥1 бонус.

Моя цель — **5/5**. Проверяю:

- [ ] Один HTML, нет локальных ассетов.
- [ ] Размер < 200 КБ (после финала — `wc -c index.html`).
- [ ] Mobile-friendly (проверяю в Playwright viewport 360×640).
- [ ] Inline SVG диаграмма архитектуры.
- [ ] Comparison table (честная, не самовосхваляющая).
- [ ] Playground с кнопкой Run.
- [ ] Timeline.
- [ ] Тёмная тема с переключателем.
- [ ] 0 стоп-слов из списка в `criteria.md`.
- [ ] Smoke.mjs 5/5 PASS.

## Технические ограничения

- **Стек**: чистый HTML + CSS + минимальный vanilla JS. Без
  сборщиков, без npm-пакетов в самом артефакте. Smoke.mjs — на
  Node + Playwright (он уже в `node_modules/` репо, видел в
  листинге корня).
- **Размер**: < 200 КБ. SVG — компактный, JS — компактный.
- **Шрифт**: системный sans-serif стек (нет веб-шрифтов).
- **Цвета**: ограниченная палитра, контраст WCAG AA. Тёмная тема
  — не «всё чёрное», а ink-on-cream / chalk-on-ink.
- **Скриншоты**: headless Chromium через Playwright в smoke.

## Non-goals

- Не пытаюсь выглядеть «умнее всех». Честно фиксирую ограничения.
- Не придумываю бенчмарки. Не пишу «MMLU 92.1%», если этого
  числа нет в моих знаниях.
- Не делаю красивую обёртку ради красивой обёртки. Каждый
  элемент либо несёт смысл, либо удаляется.
- Не скрываю, что это self-reporting. В подвале — пометка
  «Самоописание от самой модели».

## План работы

1. **PRD** — этот файл. Готово.
2. **Минимальный README** — заглушка с метаданными (тест,
   модель, дата, статус).
3. **GitHub issue** — child-issue в `serejaris/model-tests` с
   PRD-блоком и ссылкой на папку.
4. **Структура HTML** — собрать скелет: `<header>` с ASCII-figlet
   (style ANSI Shadow, как в `claude-md-writer` skill), `<main>`
   с секциями:
   - `#hero` — figlet + подзаголовок + 2-3 строки «что это».
   - `#about` — таблица «параметр / значение» (model id, vendor,
     cutoff, режимы).
   - `#architecture` — inline-SVG pipeline.
   - `#capabilities` — tabs: «Код», «Структура», «Длинный
     контекст», «Языки». В каждом — playground с кодом и
     кнопкой Run.
   - `#limits` — таблица capabilities vs limitations.
   - `#timeline` — M1 → M2 → M3 с датами и описанием.
   - `#themes` — переключатель светлой/тёмной.
   - `<footer>` — пометка про self-reporting.
5. **CSS** — две темы через `data-theme` атрибут, переключатель
   в шапке, mobile-first media queries.
6. **JS** — tabs, theme toggle, playground Run (эмулирует ответ
   через шаблон), копирование кода в буфер.
7. **Smoke.mjs** — Playwright, проверки по списку из
   `criteria.md` секция «Smoke-проверка».
8. **Скриншоты** — headless Chromium: hero, scroll-mid, активный
   tab, тёмная тема.
9. **README.md финал** — секция «Скриншоты» с `![…](screenshots/…)`
   в верхней трети, секции «Как открыть», «Что измерили», «Известные
   проблемы».
10. **Smoke прогон** — зафиксировать результат в README.
11. **Коммит + push** — `feat(model-tests): add minimax-m3/model-showcase`.
12. **Issue update** — ссылка на коммит, скрины, smoke.

  опираюсь только на то, что в открытом доступе про MiniMax как
  компанию (дата основания early 2022, миссия AGI), и на то, что
  я — M3 (значит M1, M2 предшествовали). Конкретные даты
  релизов M1/M2 — оставляю пометку «(точная дата не
  подтверждена)» или просто не указываю, если не уверен.
- **Риск: «рекурсия честности»** — мне сложно оценить себя
