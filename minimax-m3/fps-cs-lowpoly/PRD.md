# PRD — fps-cs-lowpoly

> Product Requirements Document для теста `fps-cs-lowpoly` трека
> `minimax-m3`. Написан **до начала кода** и обновляется по ходу
> итераций.

## 1. Контекст и цель

Проверить, что модель `opencode-go/minimax-m3` способна:

- По одному user-промпту с нуля поднять **запускаемый** 3D-шутер
  в браузере (cold-start, без скаффолда от пользователя).
- Итерировать по работающему проекту: добавить механику
  (full-auto, эффекты, анимации) в той же сессии, **не сломав**
  smoke-проверку.
- Писать код, который можно проверить автоматически
  (headless smoke + явные ассерты).

Артефакт — публичный, low-poly, играбельный. Не «сделано» в смысле
production-ready, а «сделано» в смысле можно запустить, играть и
увидеть все заявленные механики.

## 2. Scope

### In scope

- **3D-сцена** на Three.js, low-poly эстетика (BoxGeometry +
  `MeshLambertMaterial` + `flatShading`).
- **FPS-управление** игрока: WASD / стрелки, мышь + pointer-lock,
  прыжок, спринт.
- **Оружие** от первого лица: low-poly модель, idle sway, отдача
  (kick), full-auto очередь на LMB, перезарядка по `R`, магазин
  30 / резерв 120.
- **Импакт-эффекты** на каждый выстрел: muzzle flash (PointLight +
  аддитивная плоскость), трассер (Line) от ствола к точке
  попадания, импакт-декаль на стенах (FIFO 30 штук, fade 6с),
  искры (бокс-кубики с гравитацией, 0.25с).
- **5 ботов** с FSM: `PATROL → CHASE → ATTACK → DEAD`. Patrol по
  waypoint'ам, преследование при виде игрока, атака в shootRange,
  death-анимация плавным наклоном.
- **Hit reaction** у ботов: при попадании корпус наклоняется в
  сторону стрелка на ~0.25 рад за 0.18с, плюс head-flash эмиссией.
- **HUD**: HP, AMMO, ENEMIES counter, hit-flash overlay, win/lose
  overlay.
- **Headless smoke-тест** через Puppeteer, который проверяет все
  ключевые механики ассертами и падает, если что-то не работает.
- **Скриншоты** стартового экрана, gameplay, стрельбы и победы —
  для README и GitHub issue.

### Out of scope

- Multiplayer (никакого WebSocket / WebRTC).
- Звуки, кроме простого WebAudio-синтеза (beeps). Никаких
  загружаемых аудио-ассетов.
- Меню настроек, выбор оружия, инвентарь.
- Сохранение прогресса / localStorage.
- Touch / мобильный ввод.
- Сглаживание, тени, PBR-материалы, post-processing — намеренно
  flat, в духе CS 1.6.
- Дизайн-уровни, прогрессия, нарратив.

## 3. Пользовательские сценарии

1. **Старт**: юзер открывает страницу, видит title overlay с
   control hints, нажимает «CLICK TO PLAY», попадает в pointer-lock,
   оказывается в тёмной арене с ящиками и лампами на потолке. Видит
   HUD: HP 100, AMMO 30, ENEMIES 5.
2. **Разведка**: юзер двигается WASD, прыгает через низкие стены,
   встречает бота-патрульного. Бот пока не реагирует, пока не
   увидит игрока.
3. **Бой**: юзер наводит мышь на бота, зажимает LMB. Видит
   трассеры, вспышки, искры, импакт-декали на стенах. Бот
   отвечает стрельбой, у юзера падает HP. Убитый бот плавно
   ложится, ENEMIES counter уменьшается.
4. **Победа**: все 5 ботов мертвы → overlay «VICTORY» зелёным с
   кнопкой RESTART.
5. **Поражение**: HP юзера падает до 0 → overlay «YOU DIED» красным
   с кнопкой RESTART.

## 4. Метрики успеха

| Что измеряем | Целевое значение |
|---|---|
| `npm run build` без ошибок | PASS |
| `node smoke.mjs` | **5/5 проверок зелёные** |
| Full-auto очередь: `ammo` падает на ≥ 3 за 0.5с удержания LMB | PASS |
| Hit reaction бота при попадании: `painKick > 0` | PASS |
| Death animation: `deathProgress 0→1`, `body.rotation.x → -π/2` | PASS |
| Victory при 5/5 мёртвых: overlay показан, `g.ended === 'win'` | PASS |
| 0 runtime-ошибок в консоли headless Chrome | PASS |
| Размер бандла `dist/assets/*.js` | < 600 КБ (gzip < 150 КБ) |
| Cold-start до первого кадра | < 2с на mid-tier ноуте |

## 5. Технические ограничения

- **Стек**: Three.js 0.169, Vite 5, vanilla JS, без UI-фреймворков.
- **Без зависимостей-рантаймов**: минимум `three` + `vite`. Для
  smoke-теста — `puppeteer-core` (dev-only, не идёт в бандл).
- **Только AABB-коллизии** для игрока и стен. Боты проверяют
  raycast'ом line-of-sight, но сами не используют navmesh.
- **Без ассетов** — всё процедурно: BoxGeometry, CylinderGeometry,
  PlaneGeometry, CircleGeometry. Никаких внешних `.glb` / `.png`.
- **Совместимость**: десктопный Chrome/Firefox/Safari. WebGL
  обязателен, WebGL 2 не требуется.
- **Headless smoke должен работать** в software-растеризаторе
  (ANGLE/swiftshader) — это значит, нельзя закладываться на
  расширения WebGL, которые там не работают.

## 6. Non-goals

- Реалистичная баллистика и урон по частям тела.
- Регдолл-физика — death-анимация процедурная, не физическая.
- Реалистичный ИИ: боты не уклоняются, не ищут укрытия, не
  координируются. Достаточно «вижу → стреляю / бегу».
- Звуковой дизайн уровня AAA. WebAudio-бипы — утилитарно, чтобы
  был feedback на действия.
- Поддержка браузеров без WebGL.
- Mobile / touch.
- Многопользовательский режим.

## 7. План работы

Писался **до** начала кода по обоим промптам. Фиксирует порядок, MVP
и критерий «готово».

### 7.1. Порядок задач

Итерация 01 — cold-start:

1. Vite + Three.js проект в `minimax/` (чистая папка, ничего больше).
2. Low-poly арена 60×60: пол/стены/ящики/столбы/бочки/потолочные
   лампы.
3. FPS-контроллер игрока: WASD, мышь + pointer-lock, прыжок, спринт,
   AABB-коллизии со стенами.
4. Viewmodel оружия (low-poly коробка с прикладом и стволом), idle
   sway, kick-анимация отдачи.
5. Raycast-стрельба: одиночный выстрел по LMB, ammo 30/120, reload по
   `R`, audio-feedback.
6. ИИ ботов: FSM `PATROL/CHASE/ATTACK`, waypoint-патруль, line-of-
   sight, простая стрельба с jitter, нанесение урона игроку.
7. HUD: HP, AMMO, ENEMIES counter, hit-flash overlay, win/lose
   overlay.
8. Headless smoke `smoke.mjs` (минимум: страница рендерится, ammo
   тратится, HP падает от ботов, победа достижима).
9. `README.md` в корне теста + запись в issue.

Итерация 02 — патч механик:

10. Auto-fire через `fireHeld` флаг (mousedown/mouseup).
11. `effects.js` — пул трассеров / декалей / искр.
12. Muzzle flash в `weapon.js` (PointLight + аддитивная плоскость).
13. Hit reaction у ботов: `painKick` + `hitFromDir`, decay 0.18с.
14. Плавная death-анимация: `deathProgress 0→1` + body rotation.
15. Лёгкий наклон корпуса при беге.
16. Smoke расширен до 5 проверок: queue, effects, pain, death, victory.
17. Скрины `screenshots.mjs` (start, gameplay, firing, victory).
18. `PRD.md` (этот файл) дополнен по факту.

### 7.2. Граница MVP

- 5 ботов на арене, все умеют базовый FSM.
- Один тип оружия (low-poly SMG-стиль viewmodel).
- Smoke покрывает каждую заявленную механику.
- Headless работает в software-растеризаторе.
- Артефакт публичный, репо `serejaris/model-tests`.

**Не в MVP** (но не запрещено): мультиплеер, мобильный ввод,
система прокачки, разные типы оружия, звук, нарратив.

### 7.3. Критерий «готово»

- Все метрики успеха (раздел 4) — зелёные.
- Скрины для каждого ключевого состояния — на месте в `screenshots/`.
- `PRD.md` отражает финальный scope (раздел «Что в реальности
  получилось» ниже).
- Issue #2 закрыт, или новый issue открыт на следующий шаг.

### 7.4. Риски и план Б

| Риск | План Б |
|---|---|
| Pointer-lock не работает в headless | smoke использует `__game` dev-hook, чтобы телепортировать ботов / наносить урон напрямую |
| WebGL контекст не создаётся в headless без GPU | использовать `--use-angle=swiftshader` (уже в smoke) |
| Бот raycast ломается после рефакторинга структуры (root → body) | `intersectObjects(..., true)` recursive — этот баг уже пойман smoke-тестом в итерации 02 |
| Pointer-lock теряется при alt-tab / переключении окна | overlay «PAUSED» показывается; `fireHeld` сбрасывается на blur |
| Bots не доходят до игрока, тест победы не достижим за разумное время | dev-hook в `__game` позволяет мгновенно убить всех ботов для проверки victory-флоу |
