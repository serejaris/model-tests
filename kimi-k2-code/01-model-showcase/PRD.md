# PRD — model-showcase

## Контекст и цель

Создать одностраничный сайт-визитку про модель **Kimi K2.7 Code**, работающую через OpenCode (`opencode-go/kimi-k2.7-code`).

Цель — проверить три вещи одновременно:
1. Self-presentation: модель должна честно и конкретно рассказать про себя.
2. Фронтенд-скилл: один HTML-файл без внешних зависимостей, с диаграммами и интерактивом.
3. Анти-слоп: тексты не должны читаться как генерический ИИ-маркетинг.

## Scope

### In scope
- Один HTML-файл (`src/index.html`), самодостаточный.
- Системные шрифты, inline CSS/JS/SVG.
- Разделы: hero, «что это», архитектура/диаграмма, технические особенности, сравнительная таблица, playground с кодом, темная/светлая тема.
- Мобильная адаптивность (360 px и выше).
- Smoke-проверка (`smoke.mjs`) и скриншоты.

### Out of scope
- Бэкенд, API-вызовы, внешние CDN.
- Базовые изображения, аудио, видео.
- Реальная генерация ответов моделью в playground (только локальная эмуляция/подсветка).

## Пользовательские сценарии

1. **Посетитель открывает файл** — видит, кто выпустил модель, какая версия, для каких задач.
2. **Скроллит вниз** — видит SVG-схему пайплайна input → model → output с пояснениями.
3. **Кликает табы** — переключается между «Код», «Агенты», «Длинный контекст».
4. **Играет с playground** — вводит prompt, нажимает «Run», видит эмулированный ответ модели.
5. **Переключает тему** — сайт меняет светлую/тёмную палитру без перезагрузки.

## Метрики успеха

- [x] Один HTML-файл < 200 КБ без base64-картинок.
- [x] Открывается по `file://`, работает без сети.
- [x] На 360×640 нет горизонтального overflow.
- [x] В DOM есть хотя бы один `<svg>` и один интерактивный элемент.
- [x] В консоли нет ошибок при загрузке и взаимодействии.
- [x] Скриншоты: hero, после скролла, playground в работе.
- [x] ≤1 стоп-слово ИИ-слопа в основном тексте.

## Технические ограничения

- HTML5, CSS3, vanilla JS.
- Никаких внешних ресурсов: ни CDN, ни шрифты Google, ни изображения.
- SVG inline.
- Код подсвечивается через CSS (не через стороннюю библиотеку).

## Non-goals

- SEO, мета-теги, социальные кнопки.
- Многостраничность.
- Реальная интеграция с API модели.

## План работы

1. **PRD** — этот файл. Блокирует код.
2. **Структура HTML** — семантические секции, базовая сетка.
3. **Визуал** — CSS-переменные, тёмная/светлая тема, типографика.
4. **Диаграмма** — inline SVG архитектуры пайплайна.
5. **Интерактив** — табы, переключатель темы, playground.
6. **Контент** — факты про Kimi K2.7 Code, сравнительная таблица, сниппеты.
7. **Smoke** — `smoke.mjs` на Playwright/Puppeteer.
8. **Скриншоты** — headless Chromium, три кадра.
9. **README** — запуск, результаты smoke, скрины.
10. **Коммит** — `feat: add kimi-k2-code/01-model-showcase`.
