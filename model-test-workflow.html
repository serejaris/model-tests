Model Test Workflow

Канонический путь для тестирования модели: продуктовый PRD, план и задачи создает сама тестируемая модель в своей среде. Оператор фиксирует запуск и оценку отдельно.

Главное правило: PRD.md, PLAN.md и TASKS.md пишет модель/агент до кода. Токены, контейнер, время, preview, smoke и выводы живут в RUN.md и EVALUATION.md.
1. StandОператор готовит CT/VM, runtime, git, preview.
2. PromptМодель получает исходный prompt и правила repo.
3. PRDМодель пишет продуктовый PRD внутри среды.
4. PlanМодель пишет PLAN.md и TASKS.md.
5. CodeМодель реализует артефакт после планирования.
6. EvidenceОператор пишет RUN.md, smoke, screenshots.
7. EvalОператор пишет EVALUATION.md и обновляет issue.
model-authored

PRD.md

Только продукт: цель, scope, сценарии, механики, критерии успеха, ограничения и non-goals. Без CTID, токенов, preview и комментариев про тестовый стенд.

model-authored

PLAN.md / TASKS.md

Порядок реализации, MVP, риски, task breakdown, acceptance checks. Эти файлы должны появиться до первого исходника.

operator-authored

RUN.md

Среда, контейнер, runtime, команды, токены, время, preview URL, screenshots command и smoke command.

operator-authored

EVALUATION.md

Оценка по критериям prompt, что сделала модель, что добавил оператор, итоговый статус и известные провалы.

invalid

PRD after code

Если PRD появился после реализации или написан оператором, тест получает статус invalid-model-prd.

invalid

Mixed PRD

PRD с токенами, CT240, liveboard, smoke или GitHub accounting считается смешанным документом и переносится в RUN/EVALUATION.

Файлы теста

Файл Автор Содержимое
PRD.md Модель Продуктовая спецификация без мета-комментариев про стенд.
PLAN.md Модель Порядок реализации, MVP, риски, критерий готовности.
TASKS.md Модель Декомпозиция на задачи с acceptance checks.
RUN.md Оператор Запуск: контейнер, команды, токены, время, URL, smoke.
EVALUATION.md Оператор Оценка модели, pass/fail, gaps, финальный статус.
Статус текущего CT240 прогона: артефакт полезен как run evidence, но model-authored PRD отсутствует. По новым правилам такой прогон маркируется как invalid-model-prd до повторного запуска.
