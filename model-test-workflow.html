Model Test Workflow

Канонический путь для тестирования модели: продуктовый PRD, план и задачи создает сама тестируемая модель в своей среде. Оператор фиксирует запуск и оценку отдельно.

Главное правило: PRD.md, PLAN.md и TASKS.md пишет модель/агент до кода. Токены, контейнер, время, preview, smoke и выводы живут в RUN.md и EVALUATION.md.
1. StandОператор готовит CT/VM, runtime, git, preview.
2. PromptМодель получает исходный prompt и правила repo.
3. PRDМодель пишет продуктовый PRD внутри среды.
4. PlanМодель пишет PLAN.md и TASKS.md.
5. CodeМодель реализует артефакт после планирования.
6. EvidenceОператор пишет RUN.md, smoke, screenshots.
7. EvalОператор пишет EVALUATION.md и обновляет issue.
model-authored

PRD.md

Только продукт: цель, scope, сценарии, механики, критерии успеха, ограничения и non-goals. Без CTID, токенов, preview и комментариев про тестовый стенд.

model-authored

PLAN.md / TASKS.md

Порядок реализации, MVP, риски, task breakdown, acceptance checks. Эти файлы должны появиться до первого исходника.

operator-authored

RUN.md

Среда, контейнер, runtime, команды, токены, время, preview URL, screenshots command и smoke command.

operator-authored

EVALUATION.md

Оценка по критериям prompt, что сделала модель, что добавил оператор, итоговый статус и известные провалы.

invalid

PRD after code

Если PRD появился после реализации или написан оператором, тест получает статус invalid-model-prd.

invalid

Mixed PRD

PRD с токенами, CT240, liveboard, smoke или GitHub accounting считается смешанным документом и переносится в RUN/EVALUATION.

Git и issue gates

Gate Что проверяется Дрифт-статус при провале
before-code PRD.md, PLAN.md, TASKS.md появились до source files. invalid-model-prd
issue-open Child issue содержит model-authored PRD, W-label, parent epic, Project #4. tracking-drift
commit Commit message содержит Refs: serejaris/model-tests#N. git-drift
package RUN.md, EVALUATION.md, screenshots и smoke result на месте. evidence-drift
final README ссылается на PRD/RUN/EVALUATION, smoke и screenshots. docs-drift

Agent handoff contract

InputДословный prompt из prompts/<NN-slug>/prompt.md.
BoundaryРаботать только в своей папке теста, без чтения чужих решений до PRD.
OrderСначала PRD.md, затем PLAN.md, затем TASKS.md, затем код.
Product PRDВ PRD нет контейнера, токенов, liveboard, smoke или оценки модели.
GitКаждый commit связан с child issue через Refs:.
OutputЗапускаемый артефакт, README, screenshots, smoke command.

Анти-дрифт правила

Риск Правило Файл фиксации
Оператор написал PRD за модель PRD признается невалидным, переносится в RUN или EVALUATION. EVALUATION.md
Мета-инфраструктура попала в PRD Удалить из PRD, перенести в RUN. Продуктовая спецификация остается чистой. RUN.md
Код появился до PLAN/TASKS Тест маркируется invalid-model-prd, нужен fresh run. EVALUATION.md
Issue или Project не связаны Создать/починить epic, child, W-label, Project #4 lane. GitHub issue
Liveboard расходится с repo Сверить run_id, preview_url, stage, issue, artifact_path. RUN.md + liveboard JSON
Статус текущего CT240 прогона: артефакт полезен как run evidence, но model-authored PRD отсутствует. По новым правилам такой прогон маркируется как invalid-model-prd до повторного запуска.
