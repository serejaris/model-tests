Что это за модель

Большая языковая модель, заточенная под агентные задачи

Opus 4.8 — верхний по способностям ярус линейки Claude 4.x. Тот же ярус, что Opus 4.7 и 4.6, но новее: лучше держит длинные автономные прогоны, точнее находит баги, пишет понятнее и теплее.

Работает через единственный эндпоинт /v1/messages: текст, vision, tool use, structured output и серверные инструменты — всё это параметры одного запроса, а не отдельные API.

Контекст — 1 000 000 токенов по стандартной цене, без наценки за длинный контекст. До 128K токенов на ответ (большие ответы — через стриминг, чтобы не упереться в таймаут HTTP).

Адаптивное мышление включено как режим: модель сама решает, сколько думать на каждый запрос, и чередует размышление с вызовами инструментов.

Как устроен один запрос

Промпт → цикл «мышление ⇄ инструменты» → ответ

Один вызов messages.create может крутить внутренний цикл: модель думает, зовёт инструмент, читает результат, думает снова — пока не закончит. Глубину цикла регулирует параметр effort.

Чем сильна — конкретно

Возможности, а не лозунги

Каждый пункт — реальная фича API. Разверни, чтобы увидеть детали.

Адаптивное мышление + effort по умолчанию
Вместо фиксированного «бюджета на размышление» модель сама решает, когда и сколько думать (thinking: {type: "adaptive"}), и чередует мышление с вызовами инструментов. Объём работы регулируется через effort — пять уровней от low до max. По умолчанию high; xhigh — лучший выбор для кодинга и агентных задач.
Контекст 1M + compaction beta
Один миллион токенов входа по стандартной цене. Для долгих диалогов есть серверная компакция (compact-2026-01-12): у границы контекста история автоматически сжимается, и сессия продолжается без ручной нарезки.
Task budgets — бюджет на весь агентный цикл beta
Можно сказать модели, сколько токенов у неё на всю задачу (task_budget, минимум 20 000). Она видит обратный отсчёт и сама расставляет приоритеты и аккуратно завершает работу. Это отличается от max_tokens — жёсткого потолка на один ответ, о котором модель не знает.
Tool use, агенты и серверные инструменты
Свои инструменты (function calling) с автоматическим циклом через tool runner, плюс серверные инструменты Anthropic: web search / fetch, исполнение кода в песочнице, текстовый редактор, bash, память. Поверх — Managed Agents: Anthropic держит цикл агента и контейнер, где исполняются инструменты.
Structured outputs + prompt caching
Ответ можно жёстко привязать к JSON-схеме (output_config.format) — валидный парсимый вывод без префиллов. Кэширование промпта (минимум ~4096 токенов префикса для Opus 4.8) удешевляет повторяющийся контекст: чтение из кэша ≈ 0.1× от цены ввода.
Vision высокого разрешения и память
Картинки до 2576px по длинной стороне (≈3.75 Мп); координаты, которые возвращает модель, совпадают с пикселями 1:1 — полезно для computer use и разбора скриншотов. Модель лучше пишет и использует файловую память между сессиями.

Как её позвать

Минимальный запрос на трёх языках

Один и тот же вызов: адаптивное мышление, effort, потоковый или обычный ответ. Переключай вкладки.

showcase.py 
import anthropic

client = anthropic.Anthropic()  # ключ берётся из ANTHROPIC_API_KEY

resp = client.messages.create(
    model="claude-opus-4-8",
    max_tokens=16000,
    thinking={"type": "adaptive"},        # budget_tokens больше не нужен
    output_config={"effort": "high"},   # low | medium | high | xhigh | max
    messages=[{"role": "user", "content": "Объясни prompt caching одним абзацем."}],
)

for block in resp.content:
    if block.type == "text":
        print(block.text)
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();  // ANTHROPIC_API_KEY из окружения

const resp = await client.messages.create({

for (const block of resp.content)

Честно про API-поверхность

Что изменилось — и что вернёт ошибку

Opus 4.8 наследует поверхность Opus 4.7. Несколько привычек из старых моделей теперь дают 400 — лучше знать заранее.

✕ 400 Сэмплинг убран

temperature, top_p, top_k больше не принимаются. Поведение задаётся промптом и effort, а не температурой.

✕ 400 Фиксированный бюджет мышления

thinking.budget_tokens удалён. Только {"type": "adaptive"}; глубину регулирует effort.

✕ 400 Префилл ассистента

Нельзя «подсунуть» начало ответа последним assistant-сообщением. Для формата ответа — structured outputs или инструкция в system.

✓ Мышление по умолчанию скрыто

thinking.display по умолчанию "omitted" — блоки мышления пустые. Чтобы показать ход рассуждения, ставь "summarized".

✓ Mid-session system

Инструкции можно вставлять как role: "system" прямо в messages по ходу сессии — без инвалидации кэша (beta).

✓ Большой вывод — через стриминг

До 128K токенов на ответ, но при больших max_tokens нужен stream=True, иначе HTTP-таймаут.
