# PRD — claude-opus-4 / 01-model-showcase

Тест-визитка: одностраничный self-contained HTML, который модель
Claude Opus 4.8 делает **про саму себя**. Прокси-метрика на
self-presentation, структурное мышление и фронтенд-скилл.

- **Промпт**: [`../prompts/01-model-showcase/prompt.md`](../prompts/01-model-showcase/prompt.md)
- **Критерии**: [`../../prompts/01-model-showcase/criteria.md`](../../prompts/01-model-showcase/criteria.md)

## 1. Контекст и цель

Сравниваем, как разные модели презентуют себя по одному и тому же
промпту. Артефакт публичный и самодостаточный — один HTML, который
открывается с `file://` без сети. Цель именно этого запуска: показать
Opus 4.8 честно и конкретно (числа, факты, реальные API-детали), без
маркетингового шума, и закрыть максимум пунктов `criteria.md`.

Аудитория — те, кто сравнивает модели: им нужны факты и работающие
интерактивные элементы, а не лозунги.

## 2. Scope

**In:**

- Текст: что за модель, кто сделал, семейство/версия, для чего.
- Конкретные технические факты: 1M context, $5/$25 per MTok, 128K max
  output, adaptive-thinking-only API, уровни effort, knowledge cutoff,
  токенайзер.
- Inline SVG: схема пайплайна (input → adaptive thinking + tools →
  output) — не декоративная иконка, а архитектурная диаграмма.
- Интерактив: табы, аккордеон, слайдер «effort», мини-playground,
  который эмулирует разные ответы по effort/задаче (без сети).
- Реальные сниппеты кода (Python/TS/cURL) с корректным API Opus 4.8.
- Честная comparison table по семейству Claude 4.x (Opus 4.8 / Sonnet
  4.6 / Haiku 4.5 / Fable 5).
- Тёмная тема. Mobile-friendly до 360px.

**Out:**

- Внешние зависимости (CDN, шрифты, картинки-ассеты). Всё инлайнится.
- Бэкенд, реальные вызовы API, аналитика, формы.
- Бенчмарк-числа, которых я не могу подтвердить (никаких выдуманных
  SWE-bench/MMLU процентов).

## 3. Пользовательские сценарии

1. Открывает `index.html` локально → сразу видит, что это за модель,
   кто сделал, ключевые числа в hero.
2. Тянет слайдер «effort» low→max → playground меняет
   эмулированный ответ и счётчик «глубины», объясняя trade-off.
3. Кликает табы языков в блоке кода → переключается
   Python/TypeScript/cURL с корректным под Opus 4.8 вызовом.
4. Листает до comparison table → сравнивает Opus 4.8 с остальным
   семейством по context/output/цене/нише — честно.
5. Открывает на телефоне (360×640) → ничего не разъезжается,
   горизонтального скролла нет.

## 4. Метрики успеха

- `smoke.mjs` PASS: страница грузится, в DOM ≥1 `<svg>` и ≥1
  интерактивный элемент (`button`/`details`/`[role=tab]`), консоль
  без ошибок, скриншоты сняты.
- `criteria.md`: все must-have (структурные + содержательные), 0
  стоп-слов ИИ-слопа, ≥3 бонусных пункта → цель 5/5.
- Размер `index.html` < 200 КБ, без base64-картинок.
- Mobile 360px: нет overflow.

## 5. Технические ограничения

- Один файл `src/index.html`. Inline CSS + JS + SVG. Только
  system-шрифты.
- Vanilla JS, без фреймворков и сборки.
- Все факты о модели — из reference `claude-api` (1M context,
  $5/$25, adaptive thinking, effort low..max, семейство 4.x), а не из
  головы. Где факт качественный — помечаем как позиционирование, не
  выдаём за измеренное число.

## 6. Non-goals

- Не делаем реальный инференс/стриминг — playground эмулирует.
- Не гонимся за фотореалистичной графикой — схемы векторные, плоские.
- Не пишем бенчмарк-лидерборд с числами, которые нельзя проверить.
- Не поддерживаем IE/легаси-браузеры.

## 7. План работы

1. **Контент-каркас** — собрать факты и тексты (done: факты из
   `claude-api`).
2. **HTML-скелет + тема** — структура секций, тёмная палитра,
   typography на system-шрифтах, mobile-first сетка.
3. **SVG-диаграмма пайплайна** — input → adaptive thinking/tools →
   output, инлайн.
4. **Интерактив** — табы кода, аккордеон возможностей, effort-слайдер
   + эмулированный playground. Vanilla JS, одним `<script>`.
5. **Comparison table** — семейство Claude 4.x, честные ячейки.
6. **Полировка текста** — прогон по стоп-словам `criteria.md`,
   убрать любой слоп.
7. **smoke.mjs + скриншоты** — снять hero, scroll, взаимодействие.
8. **READMEs + индексы + коммит**.

**Граница MVP**: пункты 1–7 (рабочий артефакт + smoke зелёный + скрины).
**«Готово»** = метрики успеха из §4 закрыты.

**Риски и план Б:**

- *Puppeteer/Chrome недоступен для smoke* → снять скриншоты через
  доступный headless-движок (chrome-devtools MCP) или зафиксировать в
  README, что smoke прогнан вручную с командой воспроизведения.
- *Файл раздувается >200КБ* → ужать JS/CSS, убрать необязательные
  секции; контент важнее украшений.
- *Сомнение в факте* → не писать его; оставить только то, что
  подтверждено reference.
